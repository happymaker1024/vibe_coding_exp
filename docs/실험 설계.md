이 실험은 **논문 재현형 실험**으로 설계하는 것이 가장 적절하다. 목표는 “논문 결과를 최대한 근사 재현하고, `duration` 변수 포함/제외에 따른 성능 차이와 SHAP 해석 결과를 확인하는 것”이다. 첨부 논문은 Kaggle Bank Marketing Dataset을 대상으로 RF, GBM, XGBoost, LightGBM을 10-Fold Cross-Validation으로 비교하고, 우수 모델에 SHAP을 적용해 정기예금 가입 여부 예측 근거를 해석한 연구다.

# 6. 전처리 설계

## 기본 전처리

| 항목             | 처리 방식                 |
| -------------- | --------------------- |
| 컬럼명            | 소문자 변환, 앞뒤 공백 제거      |
| 종속변수 `deposit` | yes=1, no=0           |

| 결측치            | 존재 여부 확인 후 보고         |

## 중요한 주의점

논문은 원-핫 인코딩 후 총 48개 독립변수를 사용했다고 설명한다. 따라서 `pd.get_dummies(..., drop_first=True)`를 사용하면 논문과 다른 설계가 된다. 재현 목적이라면 `drop_first=False`로 두는 것이 타당하다.

## 논문 기반 하이퍼파라미터 후보

| 모델       | 후보값                                                                                                                                                            |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF       | `n_estimators=128`, `max_features=['sqrt', 'log2']`                                                                                                            |

scikit-learn 1.x에서 `max_features='auto'`는 완전히 제거되었으므로 후보값에 포함하지 않는다.

# 11. vibecoding 작업 순서

## Step 1. 데이터 점검

```text
paper_exp01/kaggle_data/bank/ 폴더에서 CSV 파일을 탐색하고 로드하라.
파일은 쉼표(,) 구분자이며 타깃 컬럼은 deposit이다.
데이터의 shape, 컬럼명, 결측치, 타깃 변수 deposit의 클래스 분포를 출력하라.
컬럼명은 소문자로 통일하고 앞뒤 공백을 제거하라.
```

## Step 2. 전처리

```text
deposit 컬럼을 target으로 분리하라.
yes/no 값을 1/0으로 변환하라.
문자형 변수는 one-hot encoding하되 drop_first=False로 처리하라.
전처리 후 독립변수 개수를 출력하라.
duration 포함 데이터셋과 duration 제외 데이터셋을 각각 생성하라.
```

## Step 3. 모델 정의

```text
RandomForestClassifier, GradientBoostingClassifier, XGBClassifier, LGBMClassifier를 정의하라.
모든 모델에 random_state=42를 적용하라.
논문 Table 6에서 각 모델의 최종 선택 하이퍼파라미터를 고정값으로 설정하라. GridSearchCV는 사용하지 않는다.
XGBoost는 xgboost 2.x 이상에서 use_label_encoder 파라미터를 제거하고, eval_metric='logloss'를 명시하라.
```

## Step 4. 10-Fold CV 평가

```text
StratifiedKFold(n_splits=10, shuffle=True, random_state=42)를 사용하라.
각 모델에 대해 Accuracy, Cohen’s Kappa, F1-Score를 계산하라.
F1-Score는 f1_score(average=’binary’)로 계산하라. 논문 수치와 차이가 클 경우 average=’weighted’로도 계산해 비교하라.
fold별 결과와 평균 결과를 모두 저장하라.
```

## Step 5. 논문 결과 비교

```text
논문 Table 7, Table 8의 성능값을 코드 안에 기준값으로 정의하라.
재현 결과와 논문 결과의 차이를 계산하라.
차이값은 reproduced - paper_value로 계산하라.
비교 결과를 CSV로 저장하라.
```

## Step 6. SHAP 분석

```text
duration 포함 실험에서는 최고 성능 모델 또는 LightGBM에 SHAP을 적용하라.
duration 제외 실험에서는 최고 성능 모델 또는 GBM에 SHAP을 적용하라.
summary plot을 저장하라.
원-핫 인코딩 후 실제 컬럼명을 확인한 뒤, duration, age, balance, housing_yes, poutcome_success 등 주요 변수의 dependence plot을 생성하라.
```

## Step 7. 결과 보고서 생성

```text
실험 목적, 데이터 정보, 전처리 방식, 모델 설정, 평가 결과, 논문 결과와의 차이, SHAP 해석, 재현 한계를 포함한 markdown 보고서를 생성하라.
```

## vibecoding에 넣을 핵심 작업지시문

아래 문장을 그대로 작업지시서 첫 부분에 넣으면 된다.

```text
이 프로젝트의 목적은 첨부 논문 「설명 가능한 정기예금 가입 여부 예측을 위한 앙상블 학습 기반 분류 모델들의 비교 분석」의 실험을 paper_exp01/kaggle_data/bank/ 폴더의 데이터로 재현하는 것이다. 논문과 동일하게 RF, GBM, XGBoost, LightGBM을 사용하고, 10-Fold Cross-Validation으로 Accuracy, Cohen’s Kappa, F1-Score를 평가한다. 실험은 duration 포함 데이터셋과 duration 제외 데이터셋으로 분리한다. 각 실험의 결과를 논문 Table 7, Table 8과 비교하고, 최고 성능 모델에 SHAP을 적용하여 변수 중요도와 예측 근거를 시각화한다. 모든 결과는 outputs/ 폴더에 CSV, PNG, Markdown 보고서 형태로 저장한다. random_state는 42로 고정한다.
```

이 설계대로 가면 단순 코드 실행이 아니라 **논문 재현 실험 → 결과 비교 → 설명가능성 분석 → 보고서화**까지 한 번에 구성할 수 있다.
